Google HTML/CSS Style Guide

一般样式规则

协议

引入外部CSS或者JS时不要再URL前面添加http或https这样的协议 
<!-- Not recommended -->
<script src="https://www.google.com/js/gweb/analytics/autotrack.js"></script>
<!-- Recommended -->
<script src="//www.google.com/js/gweb/analytics/autotrack.js"></script>
但是实际情况下如果不加上协议的话是不能下载文件的

一般格式化规则

缩进

用2个空格缩进，不要使用tab缩进或者tab和空格混用 
<ul>
  <li>Fantastic
  <li>Great
</ul>
.example {
  color: blue;
}

大小写

所有代码都使用小写 
<!-- Not recommended -->
<A HREF="/">Home</A>
<!-- Recommended -->
<img src="google.png" alt="Google">
/* Not recommended */
color: #E5E5E5;
/* Recommended */
color: #e5e5e5;

尾部的空白

删除尾部的空白

一般的元标签规则

编码

使用 UTF-8 (no BOM) 字符编码

注释

尽量使用注释

活动项目

对于未完成的代码使用TODO注释 
{# TODO(john.doe): revisit centering #}
<center>Test</center>
<!-- TODO: remove optional tags -->
<ul>
  <li>Apples</li>
  <li>Oranges</li>
</ul>

HTML样式规则

文档类型

使用HTML5文档类型 `` 不要关闭自闭合标签 使用`
`代替`
`

HTML有效性

尽量使用有效的HTML标签 可以使用W3C HTML validator这样的工具验证HTML文件

语义化

尽量使用有语义化的HTML标签

多媒体回退

为不能显示的多媒体提供替代的内容 
<!-- Not recommended -->
<img src="spreadsheet.png">
<!-- Recommended -->
<img src="spreadsheet.png" alt="Spreadsheet screenshot.">

结构、样式和行为分离

实体参考

尽量不要使用实体名称而使用字符来代替 
<!-- Not recommended -->
The currency symbol for the Euro is “&eur;”.
<!-- Recommended -->
The currency symbol for the Euro is “€”.

可有可无的标签

<!-- Not recommended -->
<!DOCTYPE html>
<html>
  <head>
    <title>Spending money, spending bytes</title>
  </head>
  <body>
    <p>Sic.</p>
  </body>
</html>
<!-- Recommended -->
<!DOCTYPE html>
<title>Saving money, saving bytes</title>
<p>Qed.
我认为这条不对

type Attribute

当引入CSS和JS时不要再link和script中使用type属性，因为text/css和test/javascript分别是默认的值

HTML格式化规则

一般格式

为块级元素，列表和表格另起一行，对每个子元素进行缩进 
<blockquote>
  <p><em>Space</em>, the final frontier.</p>
</blockquote>
<ul>
  <li>Moe
  <li>Larry
  <li>Curly
</ul>
<table>
  <thead>
    <tr>
      <th scope="col">Income
      <th scope="col">Taxes
  <tbody>
    <tr>
      <td>$ 5.00
      <td>$ 4.50
</table>

HTML引号

HTML属性值使用双引号 
<!-- Not recommended -->
<a class='maia-button maia-button-secondary'>Sign in</a>
<!-- Recommended -->
<a class="maia-button maia-button-secondary">Sign in</a>

CSS样式规则

CSS有效性

使用工具验证CSS有效性W3C CSS validator

ID和class命名

使用有意义的或通用的id和class名字 
/* Not recommended: meaningless */
#yee-1901 {}

/* Not recommended: presentational */
.button-green {}
.clear {}
/* Recommended: specific */
#gallery {}
#login {}
.video {}

/* Recommended: generic */
.aux {}
.alt {}

ID和Class命名风格

名称要尽量短并且有意义 
/* Not recommended */
#navigation {}
.atr {}
/* Recommended */
#nav {}
.author {}

Type Selectors

避免在id和class选择器上加上element 
/* Not recommended */
ul#example {}
div.error {}
/* Recommended */
#example {}
.error {}

尽量使用简写属性

/* Not recommended */
border-top-style: none;
font-family: palatino, georgia, serif;
font-size: 100%;
line-height: 1.6;
padding-bottom: 2em;
padding-left: 1em;
padding-right: 1em;
padding-top: 0;
/* Recommended */
border-top: 0;
font: 100%/1.6 palatino, georgia, serif;
padding: 0 1em 2em;

忽略0后面的单位

margin: 0;
padding: 0;

忽略小数点前面的0

font-size: .8em;

尽量使用简写的16进制符号

/* Not recommended */
color: #eebbcc;
/* Recommended */
color: #ebc;

前缀

在大型项目中有时会引入外部文件，这时可以使用前缀 
.adw-help {} /* AdWords */
#maia-note {} /* Maia */

使用连字符分隔id和class的多个单词

/* Not recommended: does not separate the words “demo” and “image” */
.demoimage {}

/* Not recommended: uses underscore instead of hyphen */
.error_status {}
/* Recommended */
#video-id {}
.ads-sample {}

Hacks

避免使用CSS Hack

CSS格式规则

声明顺序

按照字母顺序排序，有前缀的属性不参与排序 
background: fuchsia;
border: 1px solid;
-moz-border-radius: 4px;
-webkit-border-radius: 4px;
border-radius: 4px;
color: black;
text-align: center;
text-indent: 2em;

缩进每一个声明块

@media screen, projection {

  html {
    background: #fff;
    color: #444;
  }

}

在每一个声明后面使用分号

/* Not recommended */
.test {
  display: block;
  height: 100px
}
/* Recommended */
.test {
  display: block;
  height: 100px;
}

在每一个属性的冒号后面加一个空格

/* Not recommended */
h3 {
  font-weight:bold;
}
/* Recommended */
h3 {
  font-weight: bold;
}

在选择器和左大括号之间使用一个空格

/* Not recommended: missing space */
#video{
  margin-top: 1em;
}

/* Not recommended: unnecessary line break */
#video
{
  margin-top: 1em;
}
/* Recommended */
#video {
  margin-top: 1em;
}

有多个选择器的时候每个选择器占用一行

/* Not recommended */
a:focus, a:active {
  position: relative; top: 1px;
}
/* Recommended */
h1,
h3,
h3 {
  font-weight: normal;
  line-height: 1.2;
}

在两个规则之间使用一个空行分隔

html {
  background: #fff;
}

body {
  margin: auto;
  width: 50%;
}

对属性选择器和属性值使用单引号

/* Not recommended */
@import url("//www.google.com/css/maia.css");

html {
  font-family: "open sans", arial, sans-serif;
}
/* Recommended */
@import url(//www.google.com/css/maia.css);

html {
  font-family: 'open sans', arial, sans-serif;
}

CSS元标签规则

对样式进行分组，并添加注释

/* Header */

#adw-header {}

/* Footer */

#adw-footer {}

/* Gallery */

.adw-gallery {}

离别赠言

保证一致性 如果你写代码，花几分钟看看你周围人的代码，确定它的风格。如果他们在运算符两边使用空格，那么你也使用。If their comments have little boxes of hash marks around them, make your comments have little boxes of hash marks around them too. 关键点是使用共同的编码词汇以便人们可以关注于你说了什么而不是你怎么说。我们向全世界发布这个规则以便人们能够知道，但是你们内部的规则也很重要。如果你向文件中添加的代码和文件中已有的代码完全不同，当其他阅读这样的代码时会打乱他的节奏。避免这样做。 Revision 2.23
