git笔记

跳转到哈希值为fffff的版本，即将HEAD指向fffff
git reset --hard fffff

将本地master分支push到远程testing分支，如果远程没有testing分支，会自动创建
git push <url> master:testing

从远端克隆一个仓库，修改文件并上传到远端的流程：

git clone <url>
修改文件
git add <fileName> 将修改后的文件放到暂存区
git commit -m "message"
git push <url>

将本地已有的仓库push到远程新建的空仓库的流程：

进入本地仓库
添加远程仓库 git remote add origin <url>
git push -u origin master
将本地的master分支推送到origin主机，同时指定origin为默认主机，后面就可以不加任何参数使用git push了

如果远程仓库的版本比本地的新是push不上去的，这时需要从远程抓取新的提交与本地合并
git pull <url>
可以在仓库中使用 git remote -v 查看该仓库的对应远程url

之前出现的从远程仓库clone出错的原因以及解决方法：

原因：在global config中设置了错误的远程仓库url,运行git config --global -l命令查看全局配置
解决方法：修改global config文件git config --global -e将url删除即可 来源
